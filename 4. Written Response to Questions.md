# Problem 2. Land Type Classification

## Data Pre-processing

The 27 features show non-normal distributions (a mix of log-normal, right-skewed, and near-normal shapes), which was confirmed via EDA histograms across all three splits (see Appendix B). A standard Z-score normalisation would under-correct these skews, so sklearn's PowerTransformer (Yeo-Johnson method) was applied. The transformer was fitted exclusively on the training set and then applied to the validation and test sets to avoid data leakage (see Figure 2.1 to see the before and after using PowerTransformer). No samples were removed as the dataset contained no null values or infinite values, and the four target classes were reasonably balanced meaning no data augmentation was required.

![Figure 2.1 Feature distributions before and after PowerTransformer normalisation.](Fig2.1.png)

## Methodology

### Hyper-parameter Selection

Hyper-parameters for all three classifiers were selected via cross-validated search on the training set, with final model selection guided by validation accuracy.

**K-Nearest Neighbours (KNN):** A RandomizedSearchCV (20 iterations, 5-fold CV) was run over 

$\text{n\_neighbors} \in [1, 20]$ 

and 

$\text{weights} \in \{uniform, distance\}$

Randomised search was chosen over exhaustive grid search because the parameter space is small enough that 20 samples gives broad coverage without the computational overhead of evaluating every combination.

**Support Vector Machine (SVM):** A GridSearchCV (5-fold CV) was run across three kernel types simultaneously. The hyperparameter space $S$ is defined as the union of three disjoint sets based on the kernel type:

$S = S_{linear} \cup S_{rbf} \cup S_{poly}$

Where the parameter grids are defined as:

**Linear Kernel:**

$S_{linear} = \{ (C, \text{kernel}) \mid C \in \{0.1, 1, 10, 100, 1000\}, \text{kernel} = \text{'linear'} \}$

-*RBF Kernel:**

$S_{rbf} = \{ (C, \gamma, \text{kernel}) \mid C \in \{0.1, \dots, 1000\}, \gamma \in \{0.1, 0.01, 0.001, 0.0001\}, \text{kernel} = \text{'rbf'} \}$

-*Polynomial Kernel:**

$S_{poly} = \{ (C, d, \text{kernel}) \mid C \in \{0.1, \dots, 1000\}, d \in \{3, 4, 5, 6\}, \text{kernel} = \text{'poly'} \}$

Full grid search was used here because the multi-kernel structure made it important to evaluate all kernel-parameter combinations exhaustively. sklearn's multiclass SVC uses a one-vs-one (OvO) strategy internally, fitting C(4,2) = 6 binary classifiers that vote to produce the final prediction - effectively an ensemble of SVMs.

**Random Forest (RF):** A HalvingGridSearchCV (5-fold CV) was run over the hyperparameter space $S$, defined as the Cartesian product of the sets for each parameter:
$S = \{ (d, m, n) \mid d \in \text{max\_depth}, m \in \text{min\_samples\_split}, n \in \text{n\_estimators} \}$

Where:
- $\text{max\_depth} = \{2, 4, \text{None}\}$
- $\text{min\_samples\_split} = \{5, 10\}$
- $\text{n\_estimators} = \{25, 50, 100\}$

Successive halving was chosen to efficiently prune poorly performing configurations early, reducing total training time relative to full grid search.

Figure 2.2 shows the validation accuracy surface for the SVM's C and γ search on the RBF kernel, illustrating how performance peaks at moderate values before degrading.

### Hyper-parameter Discussion

### **K-Nearest Neighbors (KNN)**
- **`n_neighbors` = 1**: The model makes predictions based solely on the **single closest data point**. This creates a very complex decision boundary.
- **`weights` = distance**: Closer neighbors have a **stronger influence** on the prediction than neighbors further away. The weight of a neighbor is inversely proportional to its distance.

### **Support Vector Machine (SVM)**
- **`kernel` = RBF**: Uses the **Radial Basis Function**, which allows the model to create non-linear, circular, or "blob-like" decision boundaries to separate data that isn't linearly separable.

- **`C` = 100**: A **high penalty** for misclassification. The model will prioritize classifying all training points correctly.
- **`gamma` = 0.01**: Defines the **reach** of a single training point. A low value like 0.01 means each point has a "far" reach, resulting in a smoother, more generalized decision boundary.

### **Random Forest (RF)**
- **`max_depth` = 2**: Limits the trees to only **two levels deep**. This is a "shallow" forest resulting in simple logic.
- **`min_samples_split` = 5**: A node must have at least **5 data points** in it before it is allowed to split again.
- **`n_estimators` = 100**: The forest will consist of **100 individual decision trees**. Their results are averaged (for regression) or voted on (for classification) to improve overall stability.

All three classifiers achieved high cross-validation scores (KNN: 97.5%, SVM: 98.0%, RF: 98.3%), yet test accuracies are consistently 15–17 percentage points lower. This gap suggests a mild distribution shift between the CV folds and the held-out test set, amplified by the small dataset size (fewer than 200 training samples).

The SVM marginally outperforms KNN (84.5% vs. 83.9%) despite both achieving 100% training accuracy, indicating both models overfit to different degrees. The OvO ensemble nature of the SVM means individual misclassifications are decided by majority vote across 6 binary classifiers, which provides some robustness over KNN's single-nearest-neighbour decision. Random Forest achieved the lowest test accuracy (80.8%), which is likely attributable to the very shallow max_depth=2 constraint limiting each tree's representational power; despite 100 estimators, the shallow ensemble is unable to capture higher-order interactions between spectral features.

Class h (Hinoki, 15 training samples) is the most challenging class across all models, with the lowest precision (KNN: 0.59, SVM: 0.57, RF: 0.72). Its small sample count means the models have less information to establish a reliable decision boundary for this class. Class o (other non-forest) achieves the highest precision across all models (KNN: 0.96, SVM: 0.86, RF: 0.92), suggesting its spectral signature is the most spectrally distinct of the four classes.

Overall, the SVM is the recommended model: it achieves the best test accuracy and benefits from its ensemble OvO structure, which provides implicit robustness for this small, four-class problem.


# Problem 3. Training and adapting Image Classification Deep Networks

## Data Pre-processing

Images were loaded from .mat files (32×32 RGB) and normalised to [0, 1] by dividing by 255. No resizing or colour-space conversion was applied, preserving the full spatial and spectral information available at the input resolution. The training set was left in its original split (1,000 samples); class balancing for the augmented model was handled during augmentation rather than pre-processing (see Methodology).

## Methodology

### Network Design

The two models compared were a very simple model trained on non-augmented data, and a custom one of the same complexity but with varying activation and dropout trained on augmented data, inspired by in-class tutorial examples, with three paired convolutional blocks, each block consisting of two Conv2D layers followed by MaxPooling, the models are below

**Simple model:**

| Block | Filters | Kernel | Activation | Extras |
|-------|---------|--------|------------|--------|
| 1 | 8 | 3×3 | ReLU | - |
| 2 | 16 | 3×3 | ReLU | - |
| 3 | 32 | 3×3 | ReLU | - |
| FC | 64 units | - | ReLU | - |
| Output | 10 units | - | Softmax | - |

**Custom model:**

| Block | Filters | Kernel | Activation | Extras |
|-------|---------|--------|------------|--------|
| 1 | 8 | 3×3 | Swish | BatchNorm, SpatialDropout2D(0.2) |
| 2 | 16 | 3×3 | Swish | BatchNorm, SpatialDropout2D(0.2) |
| 3 | 32 | 3×3 | Swish | BatchNorm, SpatialDropout2D(0.2) |
| FC | 64 units | - | Swish | L1 kernel regularisation |
| Output | 10 units | - | Softmax | - |

Table 3.1: Network architecture summary. All Conv2D layers use same padding; each block ends with MaxPooling2D(2×2).

Swish activation was chosen over ReLU because it is smooth and non-monotonic, which has been shown to improve gradient flow. Batch normalisation after each convolutional block stabilises activations and accelerates convergence. SpatialDropout2D (rate=0.2) drops entire feature maps rather than individual neurons, which is more appropriate for spatially correlated convolutional features and acts as a stronger regulariser. L1 regularisation on the dense layer encourages sparse activations and further guards against overfitting on the small (1,000-sample) training set.

### Training Approach

All models were compiled with the Adam optimiser (lr=0.0001) and SparseCategoricalCrossentropy loss, trained for up to 500 epochs with a batch size of 128, monitored against the 1,000-sample validation set. The small batch size relative to training set size (128/1000) means each epoch sees the whole training set nearly 8 times before updating the learning rate schedule. Early stopping (patience=10, restoring best weights) was used to prevent unnecessary training once validation loss stopped improving.

The low learning rate of 0.0001 was necessary because the training set is small (1,000 samples): larger learning rates caused unstable validation loss. Each epoch takes approximately 10ms, making 500-epoch training feasible without significant compute cost.

Figure 3.1 shows training and validation accuracy and loss curves for the models. The non-augmented model converges faster initially (fewer samples per epoch) but plateaus earlier as train accuracy reaches 100% and shows a larger train/val gap indicating overfitting. The custom CNN (with BatchNorm and Dropout) converges more smoothly and achieves the best validation performance with a smaller train/val gap.

![Figure 3.1 Training and validation accuracy and loss curves for models](Fig3.1.png)

### Data Augmentation

To address the class imbalance in the training set (uneven distribution across 10 digit classes), a target count of 200 samples per class was set. For each under-represented class, existing samples were randomly selected and passed through an augmentation pipeline until the class reached 200 samples, expanding the total training set from 1000 to 2000 samples with balanced targets.

The augmentation pipeline consisted of:
- `RandomRotation(0.05)` - up to +/-18deg rotation
- `RandomZoom(0.025)` - up to +/-2.5% zoom
- `RandomTranslation(height=0.025, width=0.025)` - up to +/-2.5% translation in each axis

These transforms were selected because house number digits are roughly upright with minor perspective variation due to camera angle and distance; small rotations and translations reflect realistic real-world variation without introducing unrealistic distortions. Zoom was kept small to avoid cropping digit edges. Flipping was deliberately omitted because digit identities are not flip-invariant (e.g., flipping a 6 produces a 9).

Model 2 (Custom CNN) achieves the strongest test performance. The combination of Batch Normalisation (which reduces internal covariate shift), SpatialDropout2D (which regularises correlated spatial features more effectively than standard dropout), and Swish activation (smoother gradient flow) collectively reduce overfitting and improve generalisation compared to the Simple CNN. The L1 regulariser on the dense layer further suppresses irrelevant feature combinations.

CNN inference times are significantly faster than the SVM on the full test set because CNN forward passes are implemented as batched matrix operations on GPU/CPU BLAS kernels, whereas kernel SVM evaluation scales with the number of support vectors times the number of test samples.

## Appendix A - Iterative Log-space Search for Ridge and Lasso Regression in Problem 1.

### Ridge

```python
rmse_train_rr = []
rmse_val_rr = []
rmse_test_rr = []
coeffs_rr = []

alphas_rr = alphas.copy()
alphas_rr_history = []

for p in range(0, passes):
    start_idx = len(rmse_val_rr)
    
    for a in alphas_rr:
        rr_model = sm.OLS(y_train, X_train).fit_regularized(alpha=a, L1_wt=0)
        rmse_train_rr.append(np.sqrt(mean_squared_error(y_train, rr_model.predict(X_train))))
        rmse_val_rr.append(np.sqrt(mean_squared_error(y_val, rr_model.predict(X_val))))
        rmse_test_rr.append(np.sqrt(mean_squared_error(y_test, rr_model.predict(X_test))))
        coeffs_rr.append(rr_model.params)
        alphas_rr_history.append(a)

    pass_rmse = rmse_val_rr[start_idx:]
    min_val_idx = np.argmin(pass_rmse)
    min_val_rmse = pass_rmse[min_val_idx]
    best_alpha = alphas_rr[min_val_idx]
    
    print(f"Pass {p + 1}/{passes} result: min RMSE: {min_val_rmse} | best alpha: {best_alpha}")

    lower_idx = max(0, min_val_idx - 1)
    upper_idx = min(len(alphas_rr) - 1, min_val_idx + 1)
    
    start_val = np.log10(alphas_rr[lower_idx])
    end_val = np.log10(alphas_rr[upper_idx])
    
    alphas_rr = np.logspace(start_val, end_val, 10)

sorted_indices = np.argsort(alphas_rr_history)

alphas_rr_history = np.array(alphas_rr_history)[sorted_indices]
rmse_train_rr     = np.array(rmse_train_rr)[sorted_indices]
rmse_val_rr       = np.array(rmse_val_rr)[sorted_indices]
rmse_test_rr      = np.array(rmse_test_rr)[sorted_indices]
coeffs_rr         = np.array(coeffs_rr)[sorted_indices]
```
